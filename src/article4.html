BAB 4. PEMBAHASAN MATERI MENGENAI UML




Apa itu UML?

UML (Unified Modeling Language) adalah bahasa standar untuk memvisualisasikan, mendesain, dan mendokumentasikan sistem perangkat lunak. Dengan UML, kita bisa menggambarkan bagaimana sistem bekerja sebelum kita mulai coding.

UML membantu tim pengembang memahami:

  • 1. Alur sistem
  • 2. Hubungan antar komponen
  • 3. Interaksi antar pengguna dan sistem

Ada banyak jenis diagram di UML, tapi hari ini kita fokus pada 4 diagram penting: Use Case Diagram, Activity Diagram, Class Diagram, dan Sequence Diagram.



1. Use Case Diagram

Use Case Diagram adalah salah satu jenis diagram dalam UML yang digunakan untuk menggambarkan fungsi-fungsi utama yang dapat dilakukan dalam suatu sistem dari perspektif pengguna atau aktor. Diagram ini menyoroti interaksi antara aktor (pengguna atau sistem lain) dengan sistem yang dikembangkan. Use Case Diagram digunakan untuk memahami dan memvisualisasikan fungsionalitas sistem secara keseluruhan. Ini memberi gambaran yang jelas tentang siapa yang akan menggunakan sistem dan apa yang mereka harapkan dapat dilakukan sistem untuk mereka. Biasanya, Use Case Diagram digunakan pada tahap awal perancangan sistem untuk menetapkan apa saja yang akan dicapai oleh sistem.

Contoh Project: Perancangan sistem informasi pengaduan fasilitas berbasis web.

Perancangan Use Case Diagram:

Aktor:

  • 1. User/Pengguna (Mahasiswa/Pengunjung)
  • 2. Admin

Use Case:

User:

  • - Registrasi akun
  • - Login
  • - Buat pengaduan
  • - Lihat status pengaduan

Admin:

  • - Login
  • - Verifikasi pengaduan
  • - Tanggapi pengaduan
  • - Update status pengaduan
  • - Kelola data pengaduan

Relasi:

  • User → Registrasi, Login, Buat Pengaduan, Lihat Status
  • Admin → Login, Verifikasi, Tanggapi, Update Status, Kelola Data

2. Activity Diagram

Activity Diagram adalah diagram yang menggambarkan alur aktivitas atau proses dalam sistem. Diagram ini mirip dengan flowchart, namun lebih terstruktur dan bisa menggambarkan proses yang lebih kompleks dengan lebih banyak elemen.Activity Diagram berfokus pada alur kerja dan bagaimana aktivitas tersebut dikendalikan dalam sistem. Diagram ini berguna untuk memvisualisasikan proses bisnis atau alur kerja sistem dan memahami bagaimana tindakan pengguna atau sistem akan bergerak dalam urutan tertentu.

Contoh Project: Perancangan sistem informasi pengaduan fasilitas berbasis web.

Perancangan Activity Diagram:

Activity Diagram untuk Pengguna

  • Registrasi Akun
  • Login
  • Akses Halaman Pengaduan
  • Isi Form Pengaduan
  • Submit Laporan
  • Sistem Menyimpan Pengaduan
  • Lihat Status Pengaduan

Activity Diagram untuk Admin

  • Login
  • Verifikasi Pengaduan
  • Memberikan Tanggapan
  • Update Status Pengaduan


3. Class Diagram

Class Diagram menggambarkan struktur kelas dalam sistem. Diagram ini menampilkan kelas-kelas yang ada dalam sistem, serta atribut (data) dan method (fungsi) yang dimiliki oleh setiap kelas. Class Diagram adalah bagian integral dari desain objek dalam pemrograman berorientasi objek.Class Diagram sangat penting untuk merancang dan merencanakan struktur sistem, terutama untuk aplikasi berbasis objek. Diagram ini membantu pengembang untuk memahami bagaimana berbagai objek dalam sistem saling berinteraksi dan berfungsi.

Contoh Project: Perancangan sistem informasi pengaduan fasilitas berbasis web.

Perancangan Class Diagram:

1. User

  • id_user : int (Primary Key)
  • nama : varchar
  • email : varchar
  • password : varchar

2. Pengaduan

  • id_pengaduan : int (Primary Key)
  • id_user : int (Foreign Key ke User)
  • judul_pengaduan : varchar
  • deskripsi_pengaduan : text
  • tanggal_pengaduan : date
  • status_pengaduan : enum/varchar ('Menunggu', 'Diproses', 'Selesai')

3. Tanggapan

  • id_tanggapan : int (Primary Key)
  • id_pengaduan : int (Foreign Key ke Pengaduan)
  • id_admin : int (Foreign Key ke Admin)
  • isi_tanggapan : text
  • tanggal_tanggapan : date

4. Admin

  • id_admin : int (Primary Key)
  • nama_admin : varchar
  • username : varchar
  • password : varchar


4. Sequence Diagram

Sequence Diagram adalah salah satu jenis diagram dalam UML (Unified Modeling Language) yang menggambarkan interaksi antar objek dalam urutan waktu tertentu. Diagram ini menunjukkan bagaimana objek-objek dalam sistem berkomunikasi satu sama lain dengan saling mengirimkan pesan untuk mencapai tujuan tertentu.

Sequence Diagram sangat berguna untuk menggambarkan bagaimana sistem berfungsi secara dinamis, dan bagaimana proses atau alur tertentu diproses dalam sistem berdasarkan urutan waktu yang jelas.

Fungsi Utama Sequence Diagram:

  • Memvisualisasikan Interaksi: Sequence Diagram menggambarkan bagaimana objek dalam sistem berinteraksi satu sama lain. Interaksi ini bisa berupa panggilan fungsi, pengiriman data, atau respons balik.
  • Memahami Proses: Dengan melihat urutan pesan dalam Sequence Diagram, pengembang dan tim dapat memahami bagaimana suatu proses dilakukan dari awal hingga akhir.
  • Menangani Skenario: Sequence Diagram bisa dibuat untuk menggambarkan berbagai skenario dalam sistem, mulai dari skenario sederhana hingga skenario kompleks dengan banyak objek yang terlibat.
  • Menganalisis Kinerja Sistem: Diagram ini memungkinkan kita untuk mengevaluasi kinerja dan efisiensi dari urutan interaksi antar objek dalam skenario tertentu.

Contoh Project: Perancangan sistem informasi pengaduan fasilitas berbasis web.
